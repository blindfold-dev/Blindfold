# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Blindfold, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, please email us at **support@blindfold.dev** with:

- A description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fixes (optional)

We will acknowledge your report within 48 hours and aim to provide a fix within 7 days for critical issues.

## Supported Versions

| Package | Version | Supported |
|---------|---------|-----------|
| blindfold-sdk (Python) | 1.x | Yes |
| @blindfold/sdk (JS) | 1.x | Yes |

## Security Best Practices

- Never hardcode your API key. Use environment variables.
- Keep your SDK version up to date.
- Use HTTPS only (enforced by default).
